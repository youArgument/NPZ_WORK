# Руководство По Использованию (Load + Tests)

## Что нужно сделать?
- В файле `EntryPoint_temp.py`:
  - Реализовать pipeline инструмента;
  - Продумать момент с конвертацией (`convert_points_to_plot_coordinate`) из пиксельной системы координат -> (дистанция, углы в часах).
- В файле `schemas.py` - интегрировать в класс `SearchParams` необходимые поля (параметры для настройки поиска, от пользователя).

## Pipeline работы инструмента в рамках симуляции
1. Считывание патча (из файла `.npz`) из папки `INPUT_PATH/npz_datasets`.
2. *(Опционально)* Отрисовка магнитограммы без рамок.
3. Передача  batch (данных), params (параметры, в случае, если необходимы дополнительные поля - инициализировать их на этом шаге вручную) в `search`.
4. Внутренняя обработка, разметка и постобработка данных; передача результатов `TargetData` обратно из `search` (в координатах "дистанция-часы").
5. *(Опционально)* Загрузка данных с размеченными рамками в `OUTPUT_PATH/npz_datasets` с помощью `npz_exporter.save_to_npz`.
6. *(Опционально)* Отрисовка магнитограммы с размеченными рамками (в пиксельных координатах).


## Предоставленные данные
Вместе с шаблоном единой точки входа предоставлена выгрузка данных ВТД в формате `.npz` - датасет для экспериментов и отладки инструмента(располагаются в папке `df/`). Содержание датасета и стурктура:
- Предоставлены данные двух стендовых испытаний:
  - **Саратовский стенд 1200** от янв 2025 (в папке `stand-1_Sar-2025/`)
  - **Богородский стенд 1400** от янв 2026 (в папке `stand-2_Bog-2026/`) 
- Для каждого пропуска (протяжки/испытания) предоставлены данные двух ВИП (лежат внутри соответствующих папок пропусков):
  - Продольного намагничивания (МДР; в папке `mdr/`)
  - Поперечного намагничивания (УМДП; в папке `umdp/`)

Данные находятся по относительному пути `"df/{Stand}/{ILIT}/npz_datasets/"`. Каждый файл `.npz` представляет собой структуру с полями:
- `"magnetogram"` - матрица "сырых" данных ВТД (`shape: [M, N]`), где каждая строка - значения датчиков Холла, а по столбцам отложена дистанция (расстояние между строками равно шагу одометра);
- `"velocity"` - вектор скорости (`shape: [M]`);
- `"accelerometer"` - матрица данных акселерометров(`shape: [M, 3]`), где каждая строчка - набор показаний акселерометров по осям OX, OY, OZ соответственно, а а по столбцам отложена дистанция (расстояние между строками равно шагу одометра);
- `"orientation"` - вектор ориантации ВИП (`shape: [M]`), каждое значение это индекс датчика (элемента в строке) на 12 часов (если хотите добавить ориентацию, то соответствующий элемент должен быть по середине строки, а остальные должны соответственно смещены по кольцу);
- `"odomstep"` - скаляр, величина шага одометра в миллиметрах.

Данные были нарезаны из исходных по 5 метров с перекрытиев 2 метра. **Однако**, инструмент должен работать с любой шириной передаваемого на вход окна. 

Текущая конфигурация была выбрана из соображений одновременно **достаточного количества отдельных файлов** для экспериментов и **небольшого веса общего набора** для компактности и удобства передачи (в PLOT будет использовать окна около 100 метров с перекрытием 2-5 метров).

## Обзор предоставляемых скриптов шаблона единой точки входа

- `run_plot.py` - эмуляция работы плот: загрузка данных по `batch_idx` и параметров, вызов точки входа инструмента (функция `search`) и визуализация данных и результатов.
- `EntryPoint_temp.py` - шаблон точки вызова функции `search(...)` (единая точка входа); подразумевается, что реализация всего инструмента будет располагаться в этом файле.
- `schemas.py` - структура полей и expected shape; здесь необходимо будет интегрировать параметры метода в класс `SearchParams`.
- `tests.py` - тесты шаблона и реализации (коректность выгрузки и сохранения данных).

### 1. run_plot.py
Файл отвечает за загрузку данных из `npz_datasets` по индексу батча.

#### Основные функции
- `load_npz_data_batch(npz_datasets_dir, batch_idx)`
  - читает `data_batches/data_{batch_idx:05d}.npz`
  - возвращает `(DataBatch, SupportData, TargetData, SearchParams)`
- `plot_magnitogram(data_batch, target_batch: Optional, data_params: Optional):`
  - визуализирует данные
  - если передан параметр `target_batch` - результаты разметки, то они тоже визуализируются

#### Важно
- Путь к файлу собирается внутри функции.
- Если файла нет, выбрасывается `FileNotFoundError`.

### 2. EntryPoint_temp.py
Файл содержит `search(data: DataBatch, searchParams: TargetData)` - место для вашей основной логики при работе на инференсе.
Функция должна возвращать объект `TargetData`, данные рамок в котором должны быть переведены в систему координат `(дистанция, углы в часах)`.

Перевод точек из пиксельной системы координат в необходимую может быть произведён, например, с помощью фунции `convert_points_to_plot_coordinate`. В текущем шаблоне предоставлена базовая реализация, которая должна быть адаптирована под вид получаемых прогнозов, а также учтены пограничные случаи.


### 3. schemas.py
Содержит целевые структуры данных, с которыми работает пайплайн. Они ориентированы на интеграцию с `С#` и содержат следующие dataclass-ы:

- `DataBatch`
  - `Data`: размерность `[M, N]` (dtype = np.int16) - матрица значений датчиков Холла
  - `Velocity`: размерность `[M]` (dtype = np.int16) - скорость снаряда в каждом срезе (см/с)
  - `Accelerometer`: размерность `[M, 3]` (dtype = np.int16) - Ускорение для каждого среза (G*1000)
  - `Orientation`: размерность `[M]` (dtype = np.int16) -  массив номеров датчиков на 12 ч для каждого среза

- `SearchParams`
  - `StartDistance: float` - начало дистанции обрабатываемого участка. (В представленных данных этой информации нет для анонимизации)
  - `OdomStep: float` - шаг одометра, дле перевода из Index среза в дистанцию.

- `SupportData`
  - `thicks`: размерность `[M]` (dtype = np.int16) - толщины труб, 

- `TargetData`
  - `points`: размерность `[K, 4]` - координаты точек (пиксельная) рамки объекта (левой верхней, правой нижней). Записывается последовательно координата X1, Y1, X2, Y2. 
  Важно, что для объектов которые могут пересекать 0 часов, значение по Y1 > Y2, в ином случае такого не происходит.
  - `labels`: список (`TargetType`) длины `K` - метки (типы) объектов. 

- `TargetType`
  - enum для классов целей.
