## Реализация: ключевые детали

### Проблема: 3-канальный ввод YOLO
Магнитограмма в `.npz` хранится как один числовой слой (`int16`), но YOLO принимает 3-канальное изображение.
При обучении в `NewViewer` использовались серые PNG, которые Ultralytics превращал в 3 канала. Поэтому в `_prepare_image()` серое изображение копируется в три одинаковых канала.

### Проблема: отличие от обучающего preprocessing
Первичная версия подавала в YOLO слишком контрастную сырую магнитограмму. Это не соответствовало данным, на которых обучалась модель.
Итоговое решение повторяет `NewViewer`: вычитание медианы по каждому датчику, нормализация в [0,255], затем 3 одинаковых канала.

### Проблема: тайлинг дробил длинные объекты
Тайлинг 640×640 давал много дублей и фрагментов на длинных признаках.
Итоговое решение повторяет рабочий детектор `NewViewer`: `model.predict()` запускается по кадру/сектору целиком, с `agnostic_nms=True`.

### Конвертация координат
YOLO: `[x=col, y=row, x=col, y=row]` (pixel)
Магнитограмма: col → hours (× 12/N), row → distance (× OdomStep)
TargetData: `[dist1, hours1, dist2, hours2]`

```
res[:, 0] = points[:, 1] * OdomStep      # y(row) → distance
res[:, 2] = points[:, 3] * OdomStep      # y(row) → distance
res[:, 1] = points[:, 0] * 12 / N_cols   # x(col) → hours
res[:, 3] = points[:, 2] * 12 / N_cols   # x(col) → hours
```

## Итоговая диагностика ориентации и preprocessing

### Что было уточнено (2026-05-26)
Первичная гипотеза была в том, что достаточно просто транспонировать магнитограмму перед YOLO. После сравнения с рабочим проектом `D:\MyProjects\Work\NewViewer` выяснилось, что проблема была шире:

- модель обучалась на мягко подготовленных серых PNG, а не на сырой контрастной матрице;
- в `NewViewer` детекция выполняется по кадру целиком, без нарезки на тайлы;
- используется `agnostic_nms=True`;
- поддерживается параметр поворота кадра `rotation`;
- координаты bbox после поворота возвращаются обратно в систему отображения.

### Итоговое решение
В `Reff/EntryPoint_temp.py` текущая логика приведена к рабочей схеме `NewViewer`:

1. `_prepare_image()` принимает матрицу `[distance, sensors]`.
2. Для каждого датчика вычитается медиана по дистанции.
3. Значения приводятся к диапазону `[0, 255]`.
4. Серое изображение копируется в 3 канала для YOLO.
5. При необходимости применяется `yolo_rotation`.
6. `model.predict()` запускается по кадру/сектору целиком, без тайлинга.
7. Для NMS используется `agnostic_nms=True`.
8. `_invert_rotated_box()` возвращает bbox из повернутого пространства обратно в координаты `[distance, sensors]`.
9. `convert_points_to_plot_coordinate()` переводит bbox `[x1, y1, x2, y2]` в `TargetData`: `[dist1, hours1, dist2, hours2]`.

### Визуальная проверка
